Privacy Policy

DB Lens - SQLite & SQL Viewer

This Privacy Policy explains how DB Lens - SQLite & SQL Viewer collects, uses, stores, and shares information when you use the app. DB Lens - SQLite & SQL Viewer is designed primarily as a read-only local database viewer for Android. Most database browsing happens directly on your device.

Effective date: April 13, 2026 Platform: Android Developer: DevXSumit Labs

Short version

  • DB Lens - SQLite & SQL Viewer is built to inspect supported database files locally on your device.
  • The app is intended to open databases in read-only mode to reduce modification and corruption risks.
  • We use Firebase services for analytics, crash reporting, and feature-request submission.
  • We may integrate ads in a future release, and this policy is written to account for that future use.
  • We do not intentionally upload your database contents during normal browsing and query workflows.

1. Information We Process

Depending on how you use DB Lens - SQLite & SQL Viewer, the app may process the following types of information:

Local database data File name, file size, table names, schema, query results, and local metadata needed to provide the app's browsing features.
Recent file history Recent database references, last-opened timestamps, and limited metadata such as user version, stored locally on-device.
Query history Read-only SQL queries that you execute may be stored locally and scoped to the specific database you are using.
Feature request submissions Request title, request details, app version, device details, and optional current database context when you choose to submit feedback.
Analytics and diagnostics App events, crashes, non-fatal errors, device model, OS version, app version, Firebase app instance identifiers, and related technical usage information.

2. How We Use Information

We use processed information for the following purposes:

  • To open and inspect supported database files in read-only mode.
  • To show tables, schema, row details, foreign-key navigation, and SQL results inside the app.
  • To remember recent files, query history, and database-scoped user state.
  • To understand user flow, feature adoption, and points of friction in the app.
  • To diagnose crashes, errors, and performance issues.
  • To review feature requests and plan product improvements.
  • To support future advertising, consent, measurement, and policy compliance if ads are added to the app later.

3. Local Data Handling

DB Lens - SQLite & SQL Viewer is designed to work locally on your device. When you open a supported database file, the app processes that file locally so it can display tables, schema, rows, exports, and read-only SQL results.

Important: DB Lens - SQLite & SQL Viewer does not intentionally upload your database contents, row values, query results, or encryption keys to our servers as part of normal browsing, filtering, schema viewing, table viewing, or SQL-console use.

To improve usability, the app may keep local references such as recent files, query history, and database-scoped state on the device.

4. Encrypted Databases

If you open a supported encrypted database, the app may prompt you to enter a password or encryption key. That credential is intended to be used locally to attempt read-only access to the encrypted file.

  • Encryption credentials are not intended to be uploaded to our servers.
  • Encrypted databases are intended to remain read-only inside DB Lens - SQLite & SQL Viewer.
  • If an encrypted open attempt fails, limited technical error information may be recorded through analytics or crash tools to help diagnose the issue.

5. Firebase Services

DB Lens - SQLite & SQL Viewer may use Google Firebase services, including:

  • Firebase Analytics to understand app usage and user flow.
  • Firebase Crashlytics to detect crashes, record non-fatal issues, and improve stability.
  • Cloud Firestore to store submitted feature requests.

Depending on the Firebase service used, Google may receive technical information such as app instance identifiers, device information, usage signals, crash data, IP-related network information, app version, and operating-system details.

You can review Google's privacy practices here: Google Privacy Policy

6. Analytics Events We May Record

To understand how the app is used, DB Lens - SQLite & SQL Viewer may record events such as:

  • first-time user or first app open
  • app opened
  • database open requested, database opened, database open failed
  • encrypted database prompt shown, encrypted database opened
  • table opened, table viewer viewed
  • table filter used, table sort used, filters cleared
  • SQL console viewed, SQL autocomplete used, SQL query executed, SQL query failed
  • row details viewed
  • related table opened through foreign-key navigation
  • database exported, table exported
  • query history cleared, recent database history cleared
  • feature request submitted

These events are intended to describe app behavior and feature use. They are not intended to include full database contents or entire row payloads.

7. Feature Requests

When you choose to submit a feature request, the app may store information such as:

  • request title and request details
  • app version and app identifier
  • device manufacturer and model
  • Android SDK version
  • current database name, if included by the app context
  • server-side submission timestamp
Please avoid including sensitive information such as passwords, encryption keys, personal data, or confidential database contents in feature-request submissions.

8. Advertising and Future Ad Integration

DB Lens - SQLite & SQL Viewer may include ads in a future version of the app. If advertising is introduced, the app may use ad SDKs such as Google AdMob or another mobile advertising provider.

Current status: if your installed version of the app does not yet display ads, this section is included to prepare for future release and Play Store policy requirements.

If advertising is added, ad providers may collect or process information such as:

  • advertising identifiers
  • device and app identifiers
  • IP address and network-derived approximate location
  • ad impressions, clicks, and engagement events
  • device model, OS version, language, and performance diagnostics

If required by applicable law or platform policy, we intend to use consent prompts or equivalent privacy controls before serving personalized advertising.

9. Data Sharing

We do not sell personal information. We may share limited information only as needed to operate or improve the app, including with:

  • Google Firebase for analytics, crash reporting, and feature-request storage
  • future advertising or monetization providers, if advertising is added later
  • legal, regulatory, or governmental authorities when required by applicable law

10. Data Retention

  • Local recent files and query history: stored on your device until you clear them or uninstall the app.
  • Analytics and crash data: retained according to Firebase service settings and Google retention practices.
  • Feature requests: retained as needed for review, planning, support, and product improvement.

11. Security

We take reasonable steps to reduce unnecessary exposure of user data and to keep database viewing local where possible. However, no method of electronic transmission or storage is completely secure, and we cannot guarantee absolute security.

12. Your Choices

  • You can choose whether to open a database file in the app.
  • You can clear query history and recent database history from Settings.
  • You can choose whether to submit feature requests.
  • You can stop further app-side collection by uninstalling the app.
  • If advertising is added later, additional privacy or consent choices may become available.

13. Children's Privacy

DB Lens - SQLite & SQL Viewer is not directed to children under 13, and we do not knowingly collect personal information from children under 13. If you believe a child has provided personal information through the app, please contact us so that we can review the situation.

14. Changes to This Policy

We may update this Privacy Policy from time to time to reflect app changes, policy requirements, analytics updates, advertising integration, or service improvements. When we make changes, we will update the effective date at the top of this page.
